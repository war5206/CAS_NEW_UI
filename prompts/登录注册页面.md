# 登录注册模块

本文档需要生成四个页面：锁屏密码设置页面、确认密码页面、登录页面；

锁屏密码设置页面和确认密码页面可复用，做成一个组件（组件param为title、subtitle、下方的两个button按钮功能）

登录页面做成一个组件（组件param为title、subtitle、type（type分为guide-login和index-login，index-login登录有次数限制：3次，超过则需要输入超级管理员密码才可进入主页面））

## 锁屏密码页面

- 用户输入密码；

- 点击确认，向后端id为setOperationPassword的接口发送请求：

  ```json
  {
  	algorithmProcessId:"setOperationPassword"
  	param:{data:{adminPWD:"<用户输入>"operatePWD:"1234"}}
  }
  ```

  其中adminPWD为用户输入的，operatePWD为默认的：1234

- 后端接口返回的数据结构为

  ```json
  {
    "code": 200,
    "msg": "执行成功!",
    "data": {
      "state": "success"
    },
    "success": true
  }
  ```

  如果data中的state状态为succes，则密码设置成功，把密码值记录到一个变量中（确认密码时会用到），并跳转到确认密码页面；

- 如果点击跳过，则adminPWD默认为1357，并发送请求至后端，state为success则跳到登录页面
- 注意：输入完密码后，确认按钮才可点击

## 确认密码页面

- 用户输入密码；
- 点击下一步按钮，确认密码页面的密码值和锁屏密码设置页面的密码值进行比对，密码相同则进入登录页面，否则确认密码这四个文字换成密码错误，并提示两次密码不一致，请重新输入；（纯前端逻辑）
- 点击上一步按钮，回到锁屏密码页面重新设置密码，之前记录的密码值也要清除；
- 注意：用户输入密码后，下一步按钮才可点击；

## 登录页面

- 用户输入密码，输入完密码后自动向id为loginVerification的后端接口发送请求，请求如下：

  ```json
  {
    "algorithmProcessId": "loginVerification",
    "param": {
      "data": {
        "loginPWD": "<用户输入的密码值>"
      }
    }
  }
  ```

- 后端接口返回数据如下：

  ```json
  {
    "code": 200,
    "msg": "执行成功!",
    "data": {
      "result": {
        "state": "success",
        "message": "<超管/管理员/运维>"
      }
    },
    "success": true
  }
  ```

  - state为success，则登录成功，则把当前用户权限（超管/管理员/运维）用状态变量记录下来，然后进入系统选择与配置页面（该页面的内容先写一句话：系统选择，占着位）；
  - state为fail，则登录失败，重新输入；

- 点击忘记密码按钮，则确认密码页面的密码值和锁屏密码设置页面的密码值清除，并回到锁屏密码设置页面，重新设置密码；
